# AI-агент, що шукає роботу

Навчальний проєкт вебінару. Агент читає резюме кандидата, парсить вакансії з
jobs.dou.ua, оцінює відповідність, а далі — пише листи від імені AI-агента,
відгукується, читає відповіді роботодавця й узгоджує співбесіду в календарі.

## Як читати цей репозиторій

**Git-історія = сценарій уроку.** Кожен коміт — один логічний крок (Task).
Перемикаючись між комітами (`git checkout <hash>`), ви бачите код на відповідній
стадії, і він робочий у межах того, що вже зроблено.

- **Частина 1 — «Агент бачить світ»** (`notebooks/part1_agent_sees.ipynb`):
  `parse → reason` — резюме → профіль, вакансії → оцінений шортліст.
- **Частина 2 — «Агент діє»** (`notebooks/part2_agent_acts.ipynb`):
  `act` — лист, approval gate, відгук на DOU, IMAP-відповіді, календар.

## Архітектура (ports & adapters)

- `core/` — домен без жодного I/O: моделі (Pydantic), порти (`Protocol`),
  детермінована логіка, локальний календар.
- `connectors/` — адаптери, що торкаються світу (PDF, LLM, DOU, email, календар).
  Кожен реалізує один порт із `core/ports.py`.
- Ноутбук — диригент: оркестрація, стан і цикл агента живуть у клітинках.

Два наскрізні принципи: **`parse → reason → act`** (що можна вирішити кодом —
вирішуємо без LLM) і **human-in-the-loop** (агент пропонує — людина підтверджує).

## Запуск

Середовище: Python 3.14 + [uv](https://docs.astral.sh/uv/).

```bash
uv sync                  # встановити залежності
```

Секрети — у `.env` (шаблон у `.env.example`): ключі Gemini/Mistral, пошта агента
й App Password, куки DOU. Резюме покласти в `data/`. Деталі — у `doc/`.
