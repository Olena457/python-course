# Вебінар: обмеження та умови

## 1. Формат

- **Тип:** живий онлайн-вебінар (не запис)
- **Ведучий:** Костянтин (один спікер)
- **Структура:** до 2 зустрічей, кожна максимум 1,5 години (можливо в різні дні — підтверджено організатором)
- **Орієнтовні дати:** 25 або 30 число поточного місяця (дата гнучка, узгоджується)
- **Платформа проведення:** ITVDN

## 2. Замовники та мета

- **Замовники:** ITVDN (платформа записаних IT-курсів, itvdn.com/ua) та CBS — CyberBionic Systematics (школа програмування, edu.cbsystematics.com/ua)
- **Контактна особа:** Олег, CEO ITVDN
- **Бізнес-мета:** популяризація послуг обох платформ, залучення студентів на навчання
- **Наслідок для контенту:** вебінар має демонструвати цінність навчання (показати "ось чого можна навчитись"), а не бути закритим продуктовим демо. Потрібні природні точки для CTA на курси.

## 3. Аудиторія

- **Хто:** люди, що цікавляться програмуванням взагалі та Python зокрема
- **Заявлений рівень (від організаторів):** Middle
- **Реалістичне припущення:** змішана аудиторія — від зацікавлених початківців до middle-розробників (вебінар відкритий, рекламний)
- **Актуальні інтереси аудиторії:** AI, агенти

## 4. Тематичні рамки

- **Обов'язкове поєднання:** Python + AI
- **Початкова пропозиція організаторів:** «Парсинг 2.0: Як AI змінив збір даних (Пишемо розумний скрапер на Python)»
- **Організатори відкриті до зміни теми** за умови збереження зв'язки Python + AI
- **Зустрічна ідея спікера:** AI-агенти в автоматизації бізнесу
  - Кейс 1 — підсилення співробітників: агент для L2-підтримки (парсинг тікета → збір логів → дані з БД → 2–3 гіпотези → пропозиція рішення); ефект: 25 → 50–75 тікетів за зміну
  - Кейс 2 — заміна рутини: агент замість L1-підтримки (первинний збір запитів, типові питання, відповіді з документації)
  - Технічне ядро теми — два типи «конекторів»:
    1. вхідні дані → формат для «дорогої думаючої LLM» (відео/картинки/звук → текст, очищення, форматування)
    2. дії LLM в системі (MCP, tool use)
  - - грамотна організація «потоку мислення» агента (reasoning)

## 5. Дидактичні обмеження

- **Утримання уваги — критично:** живий формат, рекламна мета → не можна втрачати аудиторію
- **Заборонений антипатерн:** «після очевидних перетворень...» + показ 30 файлів по 150 рядків коду
- **Вимога до прикладів:** наскрізний, видимий, відтворюваний приклад; код подається порціями, які можна осягнути на льоту
- **Обсяг матеріалу:** має реально вміщатись у 2 × 1,5 год з запасом на питання та технічні паузи

## 6. Додаткові умови (підтверджено)

- **Запис:** буде; вебінар публікується на YouTube-каналі CodeUA (youtube.com/@CodeUA). Наслідки: частина 2 може посилатись на запис частини 1; контент має бути придатним і для "холодного" перегляду в записі.
- **Аудиторія онлайн:** зазвичай 70–300 слухачів; питань небагато → інтерактив треба провокувати самому (опитування в чаті, прості питання залу).
- **Промо:** спікер записує коротке відео до 1 хв для shorts; реєстрацію та промо ведуть CBS + ITVDN. Від спікера також знадобляться назва/тези для анонсу.
- **Живе демо:** середовище доступне (інтернет, API-ключі, бюджет на LLM-виклики). Запасний план — записані фрагменти демо на випадок збоїв — все одно варто мати.
- **CTA:** у ITVDN/CBS є релевантні курси для приземлення (Python, AI, автоматизація) — конкретні курси уточнити при підготовці фінальних слайдів.
- **Мова:** українська.

## 7. Похідні артефакти, які треба підготувати

1. Назва та тези/анонс для лендінгу реєстрації
2. Сценарій shorts-відео (до 1 хв)
3. Методичні матеріали + план двох зустрічей
4. Слайди, демо-код, репозиторій для учасників
5. Запасні записані фрагменти демо
6. Слайд(и) CTA з конкретними курсами ITVDN/CBS
