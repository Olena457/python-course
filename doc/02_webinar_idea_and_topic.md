

# Webinar: Constraints and Conditions

## 1. Format

- **Type:** Live online webinar (not a recording)  
- **Host:** Konstantin (single speaker)  
- **Structure:** Up to 2 sessions, each max 1.5 hours (possibly on different days — confirmed by organizer)  
- **Tentative dates:** 25th or 30th of the current month (flexible, to be agreed)  
- **Platform:** ITVDN  

## 2. Clients and Purpose

- **Clients:** ITVDN (platform for recorded IT courses, itvdn.com/ua) and CBS — CyberBionic Systematics (programming school, edu.cbsystematics.com/ua)  
- **Contact person:** Oleg, CEO of ITVDN  
- **Business goal:** Promote both platforms’ services, attract students to enroll  
- **Content implication:** The webinar should demonstrate the value of learning (“here’s what you can learn”), not be a closed product demo. Natural CTA points to courses are needed.  

## 3. Audience

- **Who:** People interested in programming in general and Python in particular  
- **Declared level (by organizers):** Middle  
- **Realistic assumption:** Mixed audience — from curious beginners to middle-level developers (open, promotional webinar)  
- **Current interests:** AI, agents  

## 4. Thematic Scope

- **Mandatory combination:** Python + AI  
- **Initial proposal (organizers):** “Parsing 2.0: How AI Changed Data Collection (Building a Smart Scraper in Python)”  
- **Organizers are open to topic changes** as long as Python + AI connection is preserved  
- **Speaker’s counter-idea:** AI agents in business automation  
  - **Case 1 — Augmenting employees:** Agent for L2 support (ticket parsing → log collection → DB queries → 2–3 hypotheses → proposed solution); effect: 25 → 50–75 tickets per shift  
  - **Case 2 — Replacing routine:** Agent for L1 support (initial request intake, FAQs, documentation-based answers)  
  - **Technical core:** Two types of “connectors”:  
    1. Input data → format for “expensive thinking LLM” (video/images/audio → text, cleaning, formatting)  
    2. LLM actions in system (MCP, tool use)  
  - Proper organization of agent’s “reasoning flow”  

## 5. Didactic Constraints

- **Attention retention — critical:** Live format, promotional goal → audience must not be lost  
- **Forbidden anti-pattern:** “After obvious transformations…” + showing 30 files of 150 lines each  
- **Requirement for examples:** End-to-end, visible, reproducible example; code presented in digestible portions  
- **Material volume:** Must realistically fit into 2 × 1.5 hours with buffer for questions and technical pauses  

## 6. Additional Conditions (Confirmed)

- **Recording:** Yes; webinar will be published on CodeUA YouTube channel (youtube.com/@CodeUA). Implication: Part 2 may reference Part 1 recording; content must also work for “cold” viewing in replay.  
- **Live audience:** Typically 70–300 viewers; few questions → speaker must provoke interaction (polls in chat, simple audience questions).  
- **Promotion:** Speaker records a short video (up to 1 min) for shorts; registration and promo handled by CBS + ITVDN. Speaker also provides title/abstract for announcement.  
- **Live demo:** Environment available (internet, API keys, budget for LLM calls). Backup plan — pre-recorded demo fragments in case of failures.  
- **CTA:** ITVDN/CBS have relevant courses (Python, AI, automation) — specific courses to be clarified for final slides.  
- **Language:** Ukrainian.

1. Title and abstract/announcement for registration landing page  
2. Script for shorts video (up to 1 min)  
3. Methodical materials + plan for two sessions  
4. Slides, demo code, repository for participants  
5. Backup recorded demo fragments  
6. CTA slide(s) with specific ITVDN/CBS courses
